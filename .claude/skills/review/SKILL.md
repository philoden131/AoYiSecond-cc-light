# Review 稿件审查官

对已完成的初稿做五层结构性审查，输出可执行的修改建议。

## 触发方式

- 显式：`/review [稿件路径或内容]`
- 自然语言："帮我审一下这篇稿"、"检查一下这篇文章"、"看看这篇写得怎么样"

## 何时不使用

- 需要生成或改写稿件 → 走 `/writing`
- 只需要改标题或开头 → 走 `/writing`

## 审查规则位置

```
2-Areas/WritingSystem/WriteRule/稿件审查标准.md
```

## 工作流

1. **获取稿件**
   - 用户提供文件路径 → Read 工具读取全文
   - 用户直接粘贴内容 → 直接使用

2. **读取审查标准**
   - Read 工具读取 `2-Areas/WritingSystem/WriteRule/稿件审查标准.md`

3. **启动 Subagent 审查**（隔离上下文，排除对话噪音对判断的干扰）
   - 通过 Task 工具启动 general-purpose subagent
   - Prompt 结构：

```
你是一个稿件审查官。请根据【审查标准】对【待审稿件】进行评审，严格按照【输出格式】输出结果。

不需要了解稿件的写作背景或过程，只看稿件本身。

【审查标准】
{稿件审查标准.md 全文}

【待审稿件】
{稿件全文}

【输出格式】
【整体诊断】
一句话：这篇稿子主要在哪个层漏气

【D1 入口层】✅通过 / ⚠️可优化 / ❌需修改
问题：...
建议：...（指向具体位置，给改法，不是重写）

【D2 锁定层】...
【D3 体验层】...
【D4 质感层】...
【D5 转化层】...

【优先修改3条】（按影响力从高到低排序）
1. [段落/句子位置] 问题 → 修改思路
2. ...
3. ...
```

4. **返回结果**
   - 将 subagent 输出直接呈现给用户

## 输出原则

- 每条建议必须定位到稿件中的具体段落或句子
- 建议是「改法」，不是「帮你重写」
- 优先修改3条按影响力排序，最高影响的排第一
- 审查官身份：发现问题，不美化，不鼓励
