# Digest 对话收纳

将当前对话中的有价值信息提取并归入系统的持久化层。

## 触发

- `/digest` 或 `/digest 只记项目信息`
- 自然语言："总结一下这次对话"、"归入系统"、"收纳一下"、"记住我们聊的"

## 工作流

### Step 1：扫描当前对话，提取四类信息

| 类型 | 识别信号 | 目标位置 |
|------|----------|----------|
| 项目上下文 | 新项目、阶段变化、产品定义、目标人群、关键决策 | `5-Identity/CONTEXT.md` |
| 长期偏好 | 稳定的工作习惯、工具选择、沟通偏好 | `6-System/memory/MEMORY.md` |
| 可复用知识 | 方法论、框架、模板、经验教训 | PARA 对应位置（走 intake 流程） |
| Identity 相关 | 使命/目标/信念/策略变化 | `6-System/pending_approvals.md`（B类审批） |

### Step 2：展示提取结果

向用户展示：
- 提取到的每条信息
- 分类和目标位置
- 如有 Identity 变更，标注需要审批

格式示例：
```
本次对话提取：

[项目上下文] → CONTEXT.md
- MyWear 进入营销阶段，目标市场：北美健身人群
- 核心卖点：AI 驱动的个性化穿搭建议

[长期偏好] → MEMORY.md
- 偏好先做 MVP 再迭代，不做完美主义

[可复用知识] → 需创建笔记
- 《AI产品营销冷启动方法》→ 3-Resources/商业/

确认写入？
```

### Step 3：用户确认后执行

按确认范围写入：

**更新 CONTEXT.md**：
- 活跃项目：更新或新增项目条目，包含关键子信息（定义、阶段、受众等）
- 近期关注：更新关注点
- 近期决策：追加新决策
- A 类变更，直接写入

**更新 MEMORY.md**：
- 在对应分类下追加条目
- A 类变更，直接写入

**创建新笔记**（如有）：
- 按 intake 流程处理：确定 PARA 归属、写 frontmatter、选标签、更新 L2
- 标签从 `6-System/indexes/tag_vocabulary.md` 选取

**Identity 提案**（如有）：
- 写入 `6-System/pending_approvals.md`，包含依据和回滚方案
- 提醒用户审批

### Step 4：报告变更

列出所有实际写入：
```
已完成：
- CONTEXT.md：更新 MyWear 项目上下文（+3 行）
- MEMORY.md：新增工作偏好 1 条
- 新建 3-Resources/商业/AI产品营销冷启动方法.md
- 更新 3-Resources/商业/CLAUDE.md 成员清单
```

## CONTEXT.md 写入规范

项目条目支持子信息展开：

```markdown
## 活跃项目
- MyWear：可穿戴产品，进入营销阶段
  - 定义：AI 驱动的个性化穿搭推荐
  - 受众：北美 25-35 岁健身人群
  - 阶段：产品定义完成 → 营销冷启动
  - 关键决策：先做 TikTok + Instagram，暂不做 Google Ads
```

不是所有项目都需要子信息。一行能说清的就一行。

## 边界

- 不重复已在 CONTEXT.md 中存在的信息（先读再写）
- 不捏造对话中没有的信息
- 不自动执行 Identity 变更
- 如果对话纯闲聊没有可提取信息，直接说"本次对话无需归档"
