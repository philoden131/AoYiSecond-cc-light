# 🚀 分布式断点开发方法论 - 项目使用指南

## �� 快速开始

### 1. 项目初始化
```bash
# 创建项目结构
mkdir DevProgress
mkdir DevProgress/[项目名]
touch DevProgress/README.md
touch DevProgress/[项目名]/current_task.md
touch DevProgress/[项目名]/next_session_guide.md
touch DevProgress/[项目名]/technical_decisions.md
touch DevProgress/[项目名]/code_locations.md
```

### 2. AI协作启动指令
```bash
# 告诉AI读取这些文件恢复上下文
请先读取 /DevProgress/README.md 和 /DevProgress/[项目名]/current_task.md，然后继续开发[功能名]
```

## ��️ 三层文档架构

### 📊 第一层：项目总览 (`DevProgress/README.md`)
**作用**：项目全局视角，技术栈、架构、环境配置
**内容模板**：
```markdown
# [项目名] 断点续开发系统

## �� 目的
为[项目名]提供分次开发的状态记录和上下文恢复系统

## 📁 文件组织
- `[项目名]/` - [具体功能]项目（进行中🔄）
- `completed_projects/` - 已完成项目归档

## �� AI使用指南
如需了解项目详情：
"请读取 /DevProgress/README.md 和 /DevProgress/[项目名]/current_task.md 查看当前状态和技术详情"

## 🔄 更新规则
1. 开发过程中实时更新current_task.md
2. 重要技术决策记录到technical_decisions.md
3. 代码变更记录到对应的changes文件
4. 开发结束前准备next_session_guide.md

## �� 项目状态概览
- [项目名]：[进度]%（包含具体功能）
- 风险等级：[低/中/高]
- 预计完成：[日期]
- 综合评分：[分数]/100
```

### �� 第二层：开发规则 (`DEVELOPMENT_RULES.md`)
**作用**：开发过程中的行为准则和约束
**内容模板**：
```markdown
# 开发规则文档

## �� 关键约束
- **零侵入原则**：新功能不破坏现有功能
- **向后兼容**：新API失败时自动回退
- **性能优先**：考虑并发和响应时间优化
- **数据格式严格**：必须符合前端期望的类型

## 🛠️ 工具使用规范
- 尽量使用 Serena MCP 进行代码分析
- 需要最新文档时使用 Context7 MCP
- 开发过程中实时更新文档状态

## �� 断点续开发机制
- 每次新会话开始时读取上下文文件
- 开发过程中实时更新current_task.md
- 重要决策记录到technical_decisions.md
- 代码位置记录到code_locations.md
```

## �� 使用流程

### 1. 项目启动
```bash
# 复制模板文件到项目根目录
cp DEVELOPMENT_METHODOLOGY.md [项目根目录]/

# 创建DevProgress文件夹结构
mkdir -p DevProgress/[项目名]
```

### 2. 初始化文档
- 根据模板填写 `DevProgress/README.md`
- 创建 `DevProgress/[项目名]/current_task.md`
- 设置初始任务和约束

### 3. 开始开发
```bash
# 告诉AI启动开发
请先读取 /DevProgress/README.md 和 /DevProgress/[项目名]/current_task.md，然后继续开发[功能名]
```

### 4. 持续更新
- 开发过程中实时更新 `current_task.md`
- 重要决策记录到 `technical_decisions.md`
- 代码位置记录到 `code_locations.md`

### 5. 项目完成
- 更新 `project_completion_summary.md`
- 移动到 `completed_projects/` 文件夹
- 准备下一个项目的 `next_session_guide.md`

## 🎉 方法论价值

这套方法论的核心价值在于：

1. **降低开发风险**：通过深度分析和向后兼容设计
2. **提高开发效率**：避免重复开发，发现导向
3. **保证代码质量**：多维度测试和性能优化
4. **支持团队协作**：断点续开发和知识传承
5. **适应AI时代**：工具集成和文档驱动开发

适用于任何技术栈和项目类型，特别适合现代AI辅助开发环境。[[CLAUDE]]
