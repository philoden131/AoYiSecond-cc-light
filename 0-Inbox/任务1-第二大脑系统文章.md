# 写"我的第二大脑系统"文章 - 零散想法

## 一、系统创建与构成

- 我的第二大脑是如何创建的
- 有哪些牌可以打：subagent、skill、hooks 等
- 自指分型系统
- 它怎么起作用
- 帮助我干嘛 → 去搭建系统框架

## 二、系统定位

1. **越用越懂我** - 通过提示词和 md
2. **越用越好用** - 积累错误经验和正确经验，以后能够复用

## 三、对话记录的价值

我在全量保存我和 AI 的对话记录，聊天记录里面包含着：

1. **我是谁**
2. **我的偏好、我的价值观**
3. **我希望 AI 如何对我、我希望得到的结果是什么**

这些内容是不管 AI/Agent 成为什么样的形态都会用得到的地方——上下文随带随走，不会被平台框定。

## 四、为什么选择 Claude Code

我把所有智力工作都集合到 Claude Code，因为目前它最好用也适合魔改。

今天这个小龙虾，明天小螃蟹——它们是很好用，但判断它们一定不是最终态，因为上手门槛太高。一定会出现一个或迭代出一个更易上手的东西。

但这些东西要用好一定有隐性的上手门槛——你的上下文。

> 如群友路不韦所说：
> ".md 文件即人格。配合自己常用的 Claude Code 这类 Coding Agent，可以很快把 AGENTS.md / TOOLS.md 这套'生产配置'跑顺，迭代起来也快；如果再有带历史记忆、个性化的就更好了：它能帮你'捏脸' SOUL.md / USER.md / IDENTITY.md，写出来更贴合你的习惯与风格。"

## 五、系统框架

### 系统要用什么
- PARA？
- ToDo？
- 搭个框架

### 系统要记住的东西

**关于我：**
1. 我是谁
2. 我的品味
3. 我在做什么
4. 一切有利于 AI 更好服务于你的信息——如果你经常问 AI 明天吃什么，那你喜欢吃辣也应该告诉他

**关于 AI：**
1. AI 是谁（人格化，交互时有奇效，尤其是长时间交流）
2. AI 应该如何服务我（工作准则）
3. 工作能力（Skill/提示词）

## 六、使用心法

假设这套系统我所能，下命令不要有先验假设：

1. **先验知识可能是陷阱**
2. **AI 能力进化很快**，要用发展的眼光看——比如 3 个月前用 AI 觉得不行，3 个月之后 AI 已经可以了，但还是带有 3 个月前的眼光在看
3. **以终为始**，fake it till make it——当它确实做不到的时候，判断是能力问题还是系统问题，这样就可以倒逼你和 AI 一起新增 Skill 和优化系统了。教他怎么用，教他怎么学，越用越好用。如果是确实现实因素做不到，再放弃
