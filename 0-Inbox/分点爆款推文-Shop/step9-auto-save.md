# Step 9：自动保存到草稿库

推文生成并通过质量检查后，自动保存到草稿库。

---

## 保存位置

```
AI_Roland/01-内容生产/推文管理系统/01-草稿库/YYYY-MM/
```

按月份组织，自动创建当月文件夹（如不存在）。

---

## 文件命名规则

**规则**：使用推文的第一句话作为文件名

**处理步骤**：
1. 提取推文的第一句话（到第一个换行符或标点符号）
2. 替换不能用于文件名的字符：
   - `/` → `_`
   - `:` → `-`
   - `?` → `？`
   - `*` → `＊`
   - `"` → `"`
   - `<` → `《`
   - `>` → `》`
   - `|` → `｜`
3. 如果超过50个字符，截断到50字符
4. 添加 `.md` 扩展名

**示例**：
- 推文第一句："体制内私车一直被公用怎么办"
- 文件名：`体制内私车一直被公用怎么办.md`

---

## 文件内容格式

```markdown
---
创作日期: YYYY-MM-DD
发布日期:
推文链接:
状态: 草稿
输入类型: A/B/C/D
跨界象限: AI/商业/人性/认知
情绪预检: A/B/C（XX分）
核心洞察: 一句话概括
---

# 推文正文

[推文内容]

---

# 数据指标

- 查看数:
- 点赞数:
- 转发数:
- 书签数:
- 评论数:
- 更新时间:

---

# 备注

创作时间：YYYY-MM-DD HH:mm
```

---

## 执行流程

1. 推文通过 Step 6 质量检查
2. 提取第一句话，生成文件名
3. 检查文件是否已存在：
   - 如果存在，在文件名后添加 `-v2`、`-v3` 等版本号
   - 如果不存在，直接创建
4. 生成完整的文件内容（包含元数据和推文正文）
5. 保存到 `01-草稿库/YYYY-MM/`
6. 输出保存路径，告知用户

---

## 输出示例

```
推文已保存到草稿库：
01-内容生产/推文管理系统/01-草稿库/2026-03/体制内私车一直被公用怎么办.md
```

---

## 注意事项

- 如果草稿库目录不存在，自动创建
- 如果当月文件夹不存在，自动创建
- 保存成功后，在 Step 7 输出中添加保存路径信息
