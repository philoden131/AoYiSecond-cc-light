# MEMORY

> 长期记忆：记录会跨会话持续有效的信息。
> 更新级别：A 类自动（Identity 相关内容需走审批）。

## 长期偏好
- 简洁直接，反对过度设计。
- 希望 AI 主动关联历史知识并给出有依据建议。

## 写作风格偏好（自媒体口播稿）
- 接地气的具体类比 > 泛泛而谈（例："把他们当百度用"）
- 情感节点要有力，有画面感（例："当你真的收到第一块钱，你会发现一切都变了"）
- 带身份感：显式用"普通人"让读者感觉「这是写给我的」
- 金句要锋利直接，不留余地（例："你懂了再多道理，也没有人直接给你发钱"）
- 结尾给愿景，不用互动 CTA（"评论区说说"）收尾
- 整体基调：内容要硬，表达要人。锋利但不高冷，激励但不鸡汤

## 写作系统文件命名规则
- 格式：`YYYYMM-DDHHMM-标题.md`（日期在前，标题在后）
- 生成文件名时用 `date +"%Y%m-%d%H%M"` 拿时间戳，不手动拼

## 长期目标
- 构建可持续自迭代的外挂大脑系统。
- 让 AI 在对话中更懂自己、并主动触发高价值链接。

## 关键约束
- 用户目标优先，系统维护为服务兜底。
- Identity 变更必须审批。
