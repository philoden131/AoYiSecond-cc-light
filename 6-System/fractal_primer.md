# Fractal Primer

> 目的：让新会话快速理解 L1/L2/L3 的最小形态，避免只知原则不知结构。

## L1 / 宪法层

- 路径：`/.claude/CLAUDE.md`
- 职责：定义服务目标、触发原则、证据契约、审批边界
- 变更触发：全局规则调整、顶层结构变动

## L2 / 目录层

- 路径：`/{目录}/CLAUDE.md`
- 职责：说明目录职责、成员边界、排查入口
- 变更触发：目录内文件增删改、职责变化、结构迁移

## L3 / 内容层

- 形态：笔记 frontmatter + 正文契约
- 职责：表达单篇内容的归属（para/domain/type/status）与用途
- 变更触发：内容定位变化、归属变化、状态变化

## 回环规则

1. 先检查 L3（内容是否还符合元数据）
2. 再检查 L2（成员清单与目录职责是否一致）
3. 最后检查 L1（全局原则是否需更新）

失败记录：`6-System/failure_snapshots/`

## 审批边界

- A 类：常规入库、索引更新、低风险记忆更新（可自动）
- B 类：Identity 或规则层变更（需审批）
- C 类：顶层结构与批量迁移（需审批）

审批队列：`6-System/pending_approvals.md`
