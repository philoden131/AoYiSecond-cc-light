# AoYi Second Brain

Claude Code 驱动的个人认知操作系统。

---

## 这是什么

一套运行在 Obsidian 仓库上、由 Claude Code 驱动的**外挂大脑系统**。它不是笔记软件，而是一个能记住你、理解你、主动帮你调用已有知识的 AI 协作层。

核心能力：
- **知识资产调用** — 对话时自动关联你积累的方法论、决策记录、学习笔记
- **上下文感知** — 记住你在做什么项目、什么阶段、偏好什么工作方式
- **自维护运转** — 标签清理、索引重建、记忆整理，系统自己跑

## 给谁用

奥一。一个人使用，所有交互通过 Claude Code 对话完成。

## 设计哲学

**更多智能，更少控制。**

不靠复杂脚本和硬编码规则管理系统，靠 prompt 和分形文档结构引导 AI 的判断力。Markdown 是记忆介质、是上下文载体、也是系统间通信协议。

## 目录结构

```
0-Inbox/        输入收集箱
1-Projects/     有截止日期的项目
2-Areas/        持续维护的责任领域
3-Resources/    可复用的知识资产
4-Archives/     已完成的归档
5-Identity/     身份画像（TELOS）— 变更需审批
6-System/       系统运行支持
.claude/        Claude Code 配置 + 系统宪法
```

## 三层文档架构

| 层级 | 位置 | 职责 |
|------|------|------|
| L1 | `.claude/CLAUDE.md` | 系统宪法 — 全局规则 |
| L2 | 各目录 `CLAUDE.md` | 局部地图 — 成员清单 |
| L3 | 笔记 frontmatter | 内容契约 — 归属定位 |

三层分形自相似：L1 是 L2 的折叠，L2 是 L3 的折叠。

## 日常使用

| 想做什么 | 怎么做 |
|----------|--------|
| 归档 Inbox 内容 | `/intake` |
| 总结对话归入系统 | `/digest` |
| 写文章 | `/writing` |
| 知识库维护 | `/maintain` |
| 审批提案 | 查看 `6-System/pending_approvals.md` |

大多数时候直接跟 Claude Code 自然语言对话即可，系统会自动关联知识、管理记忆。

## 自动化

- **启动时**：`startup.py` 输出知识库概览、待审批提醒、健康状况
- **结束时**：`stop_audit.py` 自动写入日常记忆碎片
- **索引重建**：`build_pointers.py` 生成知识指针 + 知识库健康报告（kb_health.json）
